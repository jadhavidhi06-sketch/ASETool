ASERecon v4.0 - User Manual (Modular Edition)
=============================================

1) PURPOSE
----------
ASERecon is a modular command-line security assessment utility for authorized testing.
It supports targeted workflows so the user can pick exactly which scans to run.

2) KEY CHANGES IN v4.0
----------------------
- Added nmap open-port and service scan module.
- Added nmap technology reconnaissance (NSE http scripts + fingerprints).
- Added nmap vulnerability module using `--script vuln`.
- Added theHarvester integration for domain email collection.
- Added interactive module selection menu in the banner flow.
- Results now preserve raw command output for complete auditing.

3) REQUIREMENTS
---------------
Python deps:
- requests
- beautifulsoup4
- dnspython
- python-whois
- lxml

System tools:
- nmap
- theHarvester

4) RUNNING THE TOOL
-------------------
Interactive mode (recommended):
  python asetool.py -t example.com

Non-interactive all modules:
  python asetool.py -t example.com --non-interactive

Non-interactive selected modules:
  python asetool.py -t example.com --non-interactive -m dns,email,ports,vulns

No JSON save:
  python asetool.py -t example.com --no-save

Verbose logs:
  python asetool.py -t example.com -v

5) MODULES
----------
- dns         DNS records and target IP
- whois       WHOIS registration details
- subdomains  crt.sh + common DNS checks
- email       MX/SPF/DMARC + theHarvester emails
- technology  HTTP header/HTML fingerprints + nmap http scripts
- ports       nmap full TCP port/service detection
- vulns       nmap vulnerability NSE scripts
- headers     security headers validation

6) OUTPUT STRUCTURE
-------------------
Each run stores results in JSON under `scan_results`.
For nmap and theHarvester modules, output includes:
- command string used
- return code
- stdout
- stderr
- parsed indicators (open ports, CVEs, emails)

7) TROUBLESHOOTING
------------------
- If nmap is missing: install nmap and rerun.
- If theHarvester is missing: install theHarvester and rerun.
- DNS/WHOIS rate limits may affect remote lookups.

8) AUTHORIZATION + LEGAL
------------------------
Only run scans with explicit written permission from the system owner.
Unauthorized scanning may violate law and policy.
